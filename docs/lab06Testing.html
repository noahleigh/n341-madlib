Lab 6: jQuery Mad Lib Testing

See the Mad Lib in action.

  • Valid Input

    1. Every input on the page is has the required attribute, so the form will not submit unless every input has been filled.

    2. The forms use type and pattern attributes to perform validation. For the name, adjective, and food, I used pattern="[a-zA-Z ]+" to only allow letters and spaces in the input.

    3. If the user tries to submit the form without properly matching input, the incorrect input forms will highlight red and a message will appear underneath describing the requirements.

    4. After testing, the validation works as described.

  • User Interface Elements

    1. Clicking on the Create Story button scrapes the form and concatenates a string with it based on the user selections, then slides over to the right where the output is displayed.

    2. Clicking on the Reset Forms button presents the user with a modal dialogue to confirm whether or not they wish to erase all their hard work.

      Clicking OK will close the dialogue and all the forms will be cleared to their default settings.

      Clicking Cancel will close the dialogue and no changes will be made to the form.

    3. Once the output is displayed, clicking the red bar or hitting the Enter key will slide back over to the filled input, allowing the user to make changes to their inputs.

    4. After testing in Firefox, Chrome and Internet Explorer, the user interface behaves as described.
